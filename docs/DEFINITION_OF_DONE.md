# Definition of Done (DoD)

このドキュメントは、現在リポジトリに含まれる各ツールの「完了条件」と「壊れる条件」を明文化したものです。READMEの主張は本DoDと一致させます。

## 共通前提

- 実行環境はブラウザ（JavaScript + Canvas）です。
- 生成物の保存はローカル端末上で完結します（アップロード先はユーザー操作次第）。
- 「復号可能」とは、**同一アルゴリズムを使う手順で意図したテキストが再現できること**を意味します。

---

## 1) Stego Snap (`index.html`)

### DoD（Roundtrip条件）

- 入力画像を読み込み、テキストをLSBへ埋め込み、PNGとして保存できる。
- そのPNGを同ツールのDecodeに入力したとき、ヌル終端までの文字列が一致する。
- メッセージが容量超過時はエラー表示される（黙って破損出力しない）。

### 壊れる条件（Known break conditions）

- JPEG再圧縮、SNS再エンコード、スクリーンショット等でピクセル値が変化すると復号不能/誤復号になる。
- クロップ・リサイズ・回転・色補正・フィルタ適用でビット配置が変わると壊れる。
- 画像先頭から連続埋め込みのため、同手法の解析コードがあれば検出可能。

### 非目標（Non-goals）

- 暗号学的機密性（鍵付き暗号としての安全性）は提供しない。
- 改ざん検出（完全性保証）・耐再圧縮性・フォレンジック耐性は提供しない。

---

## 2) Noise Talk (`noise.html`)

### DoD（Roundtrip条件）

- テキストをビット列化し、ヘッダー付き白黒グリッド画像として生成できる。
- 画像ヘッダーにグリッドサイズが描画される。
- 生成PNGをピクセル走査できる手段（スクリプト等）で読み取ると、入力文字列に対応するビット列が得られる。

### 壊れる条件

- スクリーンショット時のぼけ、リサイズ、圧縮でセル境界や色閾値が崩れると読取精度が下がる。
- 1文字=8bit前提のため、ASCII以外は実装依存で期待どおり復元できない場合がある。
- 目視だけでは正確な復号は難しく、機械的読み取り前提。

### 非目標

- 高い秘匿性（見ただけで構造は分かる）や暗号強度は目的外。
- OCR/視覚モデルに対する恒久的優位性は保証しない。

---

## 3) Prism Code (`prism.html`)

### DoD（Roundtrip条件）

- 入力文字列をUTF-16コード単位で16進4桁へ変換し、色タイル画像を生成できる。
- 16色パレット対応表が固定で、同じ対応表を使う復号スクリプトで入力文字列へ戻せる。
- 生成画像をローカル保存できる。

### 壊れる条件

- 圧縮・色変換・再サンプリングで色が変化すると桁誤りが発生する。
- スクリーンショット時の色管理・アンチエイリアスで境界色が混ざると復号が不安定になる。
- 文字コードの扱いはUTF-16コードユニット単位であり、実装差分がある環境では相互運用に注意が必要。

### 非目標

- 暗号鍵ベースの秘密保持は提供しない（色対応表は公開情報）。
- 誤り訂正や耐破損復元は実装していない。

---

## 4) Animated Prism (`animated-prism.html`) — Experimental

### DoD（Roundtrip条件）

- 現在のページは**実験的プロトタイプ**として扱う。
- 読み込み・UI・デコード処理は検証途中で、安定したエンコード→デコード往復はDoD対象外。

### 壊れる条件

- 実装が未完成/不整合のため、処理が途中で失敗する可能性が高い。
- 外部ライブラリやブラウザ差分の影響を強く受ける。

### 非目標

- 現時点で「圧縮耐性」「高度な誤り訂正」「本番利用可能な耐解析性」を保証しない。

---

## 運用ルール

- READMEで機能を紹介する際は、上記DoDにない能力を断言しない。
- 未完成・検証中の機能は `Experimental` と明記する。
