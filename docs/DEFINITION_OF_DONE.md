# DEFINITION OF DONE

## 手動テスト手順（Roundtrip）

### 前提
- ブラウザ: Chrome / iOS Safari
- 画像形式は **PNG** を使用すること（非PNGでは再圧縮によりLSBが壊れる可能性あり）

### テストケースA: 新方式（RGBのみ）
1. `index.html` を開く。
2. EncodeモードでPNG画像をアップロード。
3. 任意のメッセージ（日本語・英数字・絵文字を含む）を入力して `画像に埋め込む` を実行。
4. 結果表示で以下を確認する。
   - `使用量: X / Y ビット` が表示される。
   - `Alpha変更ピクセル数: 0` が表示される（Alphaが改変されていない）。
5. 生成画像をPNGで保存する。
6. Decodeモードで保存したPNGを再アップロードしてデコード。
7. 元メッセージと完全一致することを確認。
8. 上記 2〜7 を **20回連続** で実施し、すべて成功すること。

### テストケースB: 後方互換（Legacy decode）
1. 旧方式（RGBA全チャンネル書き込み）で作成済みのステガノPNGを用意する。
2. Decodeモードでアップロードしてデコード。
3. メッセージが復元され、表示上のデコード方式が `Legacy (RGBA 4ch)` になることを確認。

## 受け入れ条件
- Chrome と iOS Safari で encode → PNG保存 → 再アップロード → decode が20回連続成功。
- 新方式のencodeでAlphaチャンネルが改変されないことを確認可能（`Alpha変更ピクセル数: 0`）。
- 旧方式画像でもフォールバックでdecodeできる。
