# Limitations

本プロジェクトの生成画像は、以下の要因で容易に壊れます。共有前に理解してください。

## 画像処理パイプライン由来

- **SNS再圧縮**: JPEG再圧縮や再エンコードでLSB・色値が変化し、復号不能になる。
- **リサイズ**: 1px単位の意味を持つ表現（LSB/グリッド/色タイル）は縮小拡大で破壊される。
- **クロップ/回転**: 座標系前提のため、切り抜きや回転で位置情報が失われる。
- **色補正**: 自動補正、フィルタ、色空間変換で色マッピングが崩れる。

## ユーザー操作由来

- **スクリーンショット**: 端末依存の補間・圧縮・トーンマッピングでピクセル値が変わる。
- **メッセンジャー経由送信**: アプリ側最適化で意図せず再圧縮される。
- **ファイル形式変更**: PNG→JPEG変換は破壊的。

## ツール別の制約

### Stego Snap
- LSB改変は非常に脆弱で、少しの画質変化で壊れる。
- ヌル終端方式のため、破損時に途中で途切れる/文字化けする。

### Noise Talk
- 目視での厳密復号は困難で、実際はピクセル走査が必要。
- 8bit単位の実装都合上、ASCII中心の利用が安全。

### Prism Code
- 色の識別が前提。圧縮・色差で誤読が発生しやすい。
- 誤り訂正がないため、1桁の誤りが文字全体の破損を招く。

### Animated Prism (Experimental)
- 実験段階で仕様固定されておらず、安定動作を保証しない。
- READMEでは実験機能として扱い、本番利用前提にしない。

## 運用上の推奨

- 共有時はPNG原本を直接渡し、SNS投稿は「壊れる前提」で扱う。
- 機密情報には本ツール単体を使わず、標準暗号と併用する。
