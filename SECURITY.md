# Security Policy

## Supported Versions

Security fixes are provided only for the latest release line.

| Version | Supported |
| --- | --- |
| 1.x | ✅ |
| 0.x | ❌ |

## Reporting a Vulnerability

Please report vulnerabilities **privately** first.

- Preferred contact: open a private GitHub security advisory ("Report a vulnerability")
- Fallback contact: open an Issue with title prefix `[SECURITY]` **without exploit details**, then request a private channel

When reporting, include:
1. Affected file(s) and browser(s)
2. Reproduction steps
3. Impact assessment
4. Minimal proof-of-concept

## Disclosure Policy

- We acknowledge new reports within **7 days**.
- We aim to provide an initial triage within **14 days**.
- Coordinated disclosure is preferred; please allow time for a patch before public disclosure.

## Security Scope Notes

This repository is a **client-side toybox**. It does not claim military-grade security.
See `THREAT_MODEL.md` and `LIMITATIONS.md` for explicit non-goals and known break conditions.
